3  Data Structure

Class Objectives
  • Understand vector and tibble in R
  • Understand common data types in R

A data structure is essentially an organized way to store and access data efficiently. In R, there are many data structures. There are many data structues in R, but for the majority of your daily works, you only need to know two: vector and tibble.

Mastering data structures are usually the most challenging part of any programming language, especially if you do not have much experience in programming. As a rough analogy, these are the nouns of the language (data wrangling would be the verbs).

3.1 Vector

A vector is a one-dimensional array that can hold numeric, character, or logical data. In R, you can create a vector using the c() function. For example, to create a vector of numbers from 1 to 5, you can use the following code:

numbers <- c(1, 2, 3, 4, 5)
numbers
1
Create a vector of numbers from 1 to 5.
2
Display the numbers vector. 
[1] 1 2 3 4 5
What does c() stand for

c() stands for “combine.” It is used to combine elements into a vector. This is unfortunatley another bad design choice in R, that most other languages do not have.

3.2 Tibble

Tibble is the data structure in tidyverse. It is simply a combination of vectors.

As a simple xample,

library(tidyverse)
tibble(
    x = c(1, 2, 3),
    y = c("a", "b", "c")
)

An euivalent way of writing tibble

There is another equivalent way of writing it (which can comes handy when you need to encode it by hand)

tribble(
  ~x, ~y,
  1, "a",
  2, "b", 
  3, "c"
)

There are so many advantages of tibble compared to the classic data frame structure in R. For example, it never changes an input’s type and tell you exactly its input. As we can see, the firsr column says dbl, which means double (a numeic); and the second column says chr, which means character. It may sound trivial, but let me assure you, this is a common error ground in R and difficul;t to debug (speaking from hard-earned experience).

In addition, tibble forces a consistent thinking about data presentation. If you have worked with other people’s data, they can be extremly hard to work with. tibble, instead, forces a way to present data in a consistent way.

3.3 Our dataset: penguins

We will deal with importing datasets later (it is another headache in practice). For now, we will use the penguins dataset from the palmerpenguins package.

We will use a dataset called penguins from the palmerpenguins package. We will use this dataset throughout our course. This dataset contains measurements of penguins collected from three species: Adelie, Chinstrap, and Gentoo.

First things first, let us load the penguins dataset.

# install.packages("palmerpenguins")

data(package = "palmerpenguins")
